## Release automation

The release workflow publishes `rms-memory-mcp` to crates.io after the build matrix succeeds. Tag pushes (`v*`) publish automatically; manual dispatch requires `publish_crates=true` and an exact `expected_version`. Both paths compare the requested version to `Cargo.toml` before `cargo publish --locked` runs. Repository configuration must provide the protected `CARGO_REGISTRY_TOKEN` secret and approve the `crates.io` environment.
